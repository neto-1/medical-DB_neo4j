# medical-DB_neo4j
A playground python-Tkinter App for medical database instances creation using data from CSV files and neo4j as graphical database

# How to use
On how to use the medical database instances creation, please have a look at the 2 pdf: **Project_Documentation.pdf** and **Project_Guide_Manual.pdf**

# Privacy concern
For privacy concerns, we have deleted the doctor and patient nodes csv files. 
You can reproduce them  by creating empty csv files with the following headers and filling them with your own data.
The headers are very import for the code and should exactly follow the given order. 

# Doctor CSV file
`id,name,grade,speciality,email,contact`

1,John,Prof,Diabetologist,myemail,mycontact

# Patient CSV file
`id,name,age,weight,medical_history,diabetes_type,on_treatment,telefon`

1,Alice,28,65,Hypertension,Diabetes type 2,Yes,hercontact
