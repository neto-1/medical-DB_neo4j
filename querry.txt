As requested, we brought some adjustments on our python project. This Final version includes the following ameliorations:

The program now takes into consideration typos and return the corresponding typo error. For example: Typos when creating/deleting nodes or relation labels. Or creating relations between nodes that does not exist in the database.
The program initialises itself from the previous state of the database. So, you can start with any task you want on the GUI such as directly deleting a node or creating a relation between nodes that were previously created from past working sessions. There is some sort of initialisation of the current database state from most recent state. Hence each logging session is just a continuation from the preivous one, unless the database has been droped.
A new pretty way of displaying query results in tabular format on both Admin and User interfaces.
